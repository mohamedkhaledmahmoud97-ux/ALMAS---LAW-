# Repository Governance

## Ownership

- Maintainers are responsible for review, integrations, release readiness, and security oversight.

## Review Process

- All changes to protected branches must go through pull requests.
- Reviews should focus on correctness, security, architecture fit, and documentation quality.

## Approval Workflow

1. Open a pull request from a feature branch.
2. Ensure required checks pass.
3. Obtain approval from at least one maintainer.
4. Merge only after review and documentation confirmation.

## Decision Making

- Major architecture or security decisions should be recorded in ADRs.
- Cross-cutting design proposals should be discussed in GitHub Discussions.
