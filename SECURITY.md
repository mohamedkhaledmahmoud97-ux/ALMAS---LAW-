# Security Policy

## Supported Versions

This project is currently in the foundation and repository-setup phase. Security updates are applied as the project evolves.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Report suspected vulnerabilities privately by emailing the maintainers through the GitHub repository contact options or by opening a confidential security advisory if available.

## Expectations

- Do not disclose details publicly before a fix is available.
- Provide reproduction steps, impact, and affected versions when possible.
- Coordinate disclosure with the maintainers.

## Response Timeline

Maintainers will review reports as soon as possible and aim to provide an initial response within a reasonable timeframe.
